Subjects

A 'subject' is a specialized type of observable that allows you, as the developer, to control how the data is emitted.

Since an observable is emitting data, 'pipe' allows you to affect the data that is being emitted.

Looking at the observable from earlier:

const obs = from([1, 2, 3, 4]);

Once one subscribes, all 4 values will be emitted immediately. There is no way to add a new value, nor is it possible to change 'when' the values are emitted.

When you create a subject, it exposes the ability to emit through the 'next' method, and to complete any subscriptions through the complete method.

It's important to note that the Subject can emit data whether or not there are any subscriptions. This will have different effects depending on the type of subject.

Example:

const subj = new Subject<number>();
let counter = 0;
setInterval(() => {
  subj.next(++counter);
}, 1000);

const btn1 = document.querySelector('#click_me1');
const res1 = document.querySelector('#result1');
let sub: Subscription;
btn1.addEventListener('click', (e) => {
  res1.textContent = 'OBSERVING';
  const onNext = (n) => {
	res1.textContent += '\n' + `OBSERVED: ${n}`;
  };
  const onError = (e) => {
	alert('ERROR');
  };
  const onComplete = () => {
	res1.textContent += '\nCOMPLETE';
  };
  sub = subj.subscribe(onNext, onError, onComplete);
});

const clear1 = document.querySelector('#clear1');
clear1.addEventListener('click', (e) => {
  sub.unsubscribe();
  res1.textContent += '\nSTOPPED';
});
	




Some things to note


    
	
  1. The onComplete function is never called, because the 'complete' method on the subject is never called.
    2. 
	
  2. The subject is emitting vales even when there are no subscriptions.
    3. 
	
  3. Re-subscribing to the subject doesn't pick up where it left off, but at whatever the current state is.
