PART 5 - Pipes

The RxJS library supplies a 'pipe' method with any observable.

Since an observable is emitting data, 'pipe' allows you to affect the data that is being emitted.

One can:

  1. Filter the data so that only some types of data are emitted.
  2. Alter the data before it is emitted.
  3. Change the data to a complete different type
  4. And many
  5. Many
  6. Many
  7. Many
  8. Other
  9. Things

Arguably, pipes are the most complicated AND the most powerful aspects of observables.

It's important to note that when you pipe an observable, you're making a NEW OBSERVABLE.

Once could subscribe to the original observable and get different results than subscribing to the piped observable.

pipe(filter) example:

const obs1 =
  from([1, 2, 3, 4, 5])
  .pipe(filter(num => num % 2 === 0));

const btn1 = document.querySelector('#click_me1');
const res1 = document.querySelector('#result1');
btn1.addEventListener('click', (e) => {
  res1.textContent = 'OBSERVING';
  obs1.subscribe((n) => {
    res1.textContent += '\n' + `OBSERVED: ${n}`;
  });
});
